# 哈希表 / 堆 / 栈 / 队列（核心考点）

## 哈希表（Hash）
- 关注点：冲突处理、负载因子、扩容代价、最坏复杂度退化。
- 高频题型：两数之和、最长连续序列、频次统计、滑窗去重。
- 追问方向：何时用 `HashMap` 而不是有序结构；内存占用与稳定性权衡。

## 堆（Heap / Priority Queue）
- 关注点：TopK、动态中位数、合并多路有序流、最小/最大堆转换。
- 高频题型：前 K 高频元素、数据流中位数、会议室调度。
- 追问方向：为什么用堆而不是全量排序；时间复杂度从 `O(n log n)` 到 `O(n log k)` 的变化。

## 栈（Stack）
- 关注点：单调栈、括号匹配、表达式求值、回溯状态管理。
- 高频题型：每日温度、柱状图最大矩形、最小栈。
- 追问方向：入栈出栈不变式、为什么能做到线性时间。

## 队列（Queue / Deque）
- 关注点：BFS、滑动窗口、双端队列维护候选集。
- 高频题型：滑动窗口最大值、层序遍历、最短路径变种。
- 追问方向：为什么要双端队列；如何证明队头元素始终有效。

## 统一口述模板（无编码环境）
1. 先定义状态与目标。
2. 说明为什么选该数据结构。
3. 口述关键步骤（可用伪代码级描述）。
4. 给出时间/空间复杂度。
5. 给出至少 2 个边界案例（空输入、重复值、极端规模等）。
